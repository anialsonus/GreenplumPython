Table

This module creates a Python object Table which keeps in memory all the user modifications on a table, in order to proceed with SQL query. It concatenates different pieces of queries together using CTEs.

Table sends the aggregated SQL query to the database and returns the final result only when user calling fetch() function.

All modifications made by users are only saved to the database when calling the save_as() function.

Returns

  • a Column of the current Table if key is string

id_col = tab["id"]

  • a new Table from the current Table per the type of key:

    • if key is a list, then SELECT a subset of columns, a.k.a. targets;

    id_table = tab[["id"]]

    • if key is an Expr, then SELECT a subset of rows per the value of the Expr;

    id_cond_table = tab[tab["id"] == 0]

    • if key is a slice, then SELECT a portion of consecutive rows

    slice_table = tab[2:5]

apply(func: Callable[[Table], FunctionExpr]) → FunctionExpr

Apply a function to the Table

Parameters

func – Callable[[Table], FunctionExpr]: a lambda function of a FunctionExpr

Returns

a callable

Return type

FunctionExpr

Example

rows = [(i,) for i in range(-10, 0)]
series = gp.values(rows, db=db, column_names=["id"])
abs = gp.function("abs", db=db)
result = series.apply(lambda t: abs(t["id"])).to_table().fetch()

If we want to give constant as attribute, it is also easy to use. Suppose label function takes a str and a int:

result = series.apply(lambda t: label("label", t["id"])).to_table().fetch()
